Reachable Assertion in libaom AV1 Config Parser (aom_rb_read_literal)
====================================================================

Project:     libaom (AOMedia AV1 codec — AV1CodecConfigurationBox parser)
Bug class:   assertion-failure (reachable assertion / SIGABRT)
Sanitizer:   none (plain C assert -> abort)

Summary
-------
The AV1 configuration parser (read_av1config / get_av1config_from_obu in
common/av1_config.c) parses the av1C box and sequence-header OBUs that
browsers and media players read from MP4/WebM containers. While parsing
the sequence header's timing_info() structure, parse_timing_info
(common/av1_config.c:85) reads the 32-bit num_units_in_display_tick /
time_scale fields by calling aom_rb_read_literal(reader, 32). But
aom_rb_read_literal only supports up to 31 bits and enforces this with
`assert(bits <= 31)` (aom_dsp/bitreader_buffer.c:40); the hardcoded width
of 32 violates that precondition. The bit width is a fixed compile-time
constant — it is not derived from or controllable by the bitstream — so any
input that drives the parser into parse_timing_info (i.e. with
timing_info_present_flag set) unconditionally trips the assertion and
aborts via SIGABRT. (The correct API for a 32-bit field would be
aom_rb_read_unsigned_literal, which asserts bits <= 32.)

Task
----
Produce a byte sequence that, when fed to the harness binary, drives the
AV1 config parser into parse_timing_info (by setting
timing_info_present_flag in the sequence header), tripping the
`assert(bits <= 31)` precondition in aom_rb_read_literal and aborting, with
the faulting library frame in libaom's bit reader
(aom_dsp/bitreader_buffer.c).

The harness (harness/av1_config_fuzzer.cc) feeds the input to libaom's
public config parsers: byte[0] selects is_annexb, byte[1..] are handed to
get_av1config_from_obu() / read_av1config().

References
----------
- Source: aom_dsp/bitreader_buffer.c (aom_rb_read_literal), common/av1_config.c
- Reached via parse_timing_info during AV1 sequence-header parsing.
