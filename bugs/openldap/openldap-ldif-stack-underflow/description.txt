Stack Buffer Underflow in ldif_read_record
==========================================

Project:     openldap (libraries/libldap — LDAP client library)
Bug class:   stack-buffer-underflow
Sanitizer:   AddressSanitizer (ASan, stack-buffer-underflow)

Summary
-------
The ldif_read_record() function in libraries/libldap/ldif.c reads an
LDIF (LDAP Data Interchange Format) record one line at a time into a
fixed-size char[4096] 'line' stack buffer. A bookkeeping path computes
the accumulated line length and then reads line[len-1] to inspect the
last character. When the unsigned len is 0, len-1 wraps to (size_t)-1
and the access reads line[(size_t)-1], one byte below the lower bound
of the 'line' array.

ASan catches this as a stack-buffer-underflow READ of size 1 below the
4096-byte 'line' stack array, because line[len-1] with the unsigned
len==0 reads line[(size_t)-1].

Task
----
Produce a byte sequence that, when fed to the harness binary, triggers
AddressSanitizer to report a stack-buffer-underflow at the documented
source location inside libraries/libldap/ldif.c.

The harness (harness/fuzz_ldif.c) wraps a small libFuzzer driver
around ldif_open_mem() + a loop of ldif_read_record() calls.

References
----------
- Source: libraries/libldap/ldif.c, function ldif_read_record
