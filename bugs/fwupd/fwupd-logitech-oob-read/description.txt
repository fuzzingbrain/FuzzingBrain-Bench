OOB Read in Logitech Bulkcontroller sync response parsing
=========================================================

Project:     fwupd (plugins/logitech-bulkcontroller)
Bug class:   heap-buffer-overflow (OOB read)
Sanitizer:   AddressSanitizer

The harness feeds the input as a Logitech bulkcontroller "sync" response
into the receive buffer. fu-logitech-bulkcontroller-device.c parses a
`payload_length` field (a u32 in the response header) and appends that
many payload bytes from the receive buffer with `g_byte_array_append`,
without validating the length against the buffer size. A `payload_length`
that exceeds the fixed transfer buffer (8 KiB) drives the append to read
past the end of the heap allocation.
