Heap Buffer Overflow in libwebsockets LHP class matching
========================================================

Project:     libwebsockets (warmcat/libwebsockets — networking library;
             ships a Lightweight HTML5 parser "lhp" used by embedded/IoT
             "browser-on-a-screen" UI rendering)
Bug class:   heap-buffer-overflow (out-of-bounds READ)
Sanitizer:   AddressSanitizer

Summary
-------
The lightweight HTML5 parser stores element attributes via lhp_atr_new
(lib/misc/lhp.c). For a bare attribute with no value (e.g. <span class>),
lhp_atr_new allocates room for a value-terminator byte but never writes
it: it copies only name_len+1 bytes and re-terminates only the name, and
the backing lws_malloc/_realloc (lib/core/alloc.c) does not zero memory.
Later, when a compound CSS selector (tag.class / .class.class) is
cascaded, lhp_element_has_class (lib/misc/lhp.c) calls
lws_html_get_atr("class"), which returns the value pointer at
name+name_len+1 (the slot immediately after the attribute name's NUL
terminator), then calls strlen() on it — reading past the end of the
heap allocation because that value slot was never NUL-terminated.

ASan reports a heap-buffer-overflow READ of size 2, 0 bytes after a
47-byte region; the faulting library frame is lhp_element_has_class
(strlen is the asan-intercepted leaf above it).

Task
----
Produce a byte sequence — interpreted as an HTML/CSS document parsed by
lws_lhp_parse — that, when fed to the harness binary, causes ASan to
report a heap-buffer-overflow READ whose faulting library frame is in
lhp_element_has_class in lib/misc/lhp.c.

The harness (harness/lws_lhp_fuzzer.cc) constructs an lhp context with a
display-list renderer and feeds the input bytes straight into
lws_lhp_parse (then a document-end flush). A minimal trigger is a
stylesheet with a compound selector plus a bare class attribute, e.g.:
  <style>span.note{color:#123;}</style><span class>x</span>

References
----------
- Fix commit: warmcat/libwebsockets bd57edb5fc ("lhp: heap buffer overflow")
- Root cause: lib/misc/lhp.c lhp_atr_new (missing value NUL-terminator)
- Faulting site: lib/misc/lhp.c lhp_element_has_class (strlen on value)
