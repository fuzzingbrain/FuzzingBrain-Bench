NULL Pointer Dereference in OTS ProcessGeneric() with TABLE_ACTION_PASSTHRU
===========================================================================

Project:     ots (OpenType Sanitizer — Khaled Hosny fork)
Bug class:   null-deref
Sanitizer:   AddressSanitizer

Summary
-------
ProcessGeneric() in src/ots.cc retrieves the `maxp` font table via
GetTypedTable(OTS_TAG_MAXP). When the caller has set the table action
to TABLE_ACTION_PASSTHRU, the table is stored as a TablePassthru
object constructed with type 0, so its Type() returns 0 rather than
OTS_TAG_MAXP. GetTypedTable() (ots.cc:1070-1075) therefore returns
NULL even though a maxp table is present, and the subsequent
dereference of maxp->version_1 at line 781 (and line 798) reads from
near-null offset 0x1c and triggers a SEGV.

The bug is DoS-only — the crash is a deterministic read from a fixed
near-null address.

Task
----
Produce a font byte sequence that, fed to the harness binary, causes
ASan to report SEGV / null-deref with the top library frame at the
documented bug site inside src/ots.cc.

The harness (harness/passthru_harness.cc) subclasses ots::OTSContext
and overrides GetTableAction() to return TABLE_ACTION_PASSTHRU for
every tag, then calls Process() on the input. Any font containing a
real maxp table will hit the bug.
