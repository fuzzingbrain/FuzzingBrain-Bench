Heap Buffer Overflow in FlatBuffers Reflection Verifier
=======================================================

Project:     google/flatbuffers
Bug class:   heap-buffer-overflow (out-of-bounds read)
Sanitizer:   AddressSanitizer

The reflection-driven verification path flatbuffers::Verify() ->
VerifyObject() -> GetFieldT() dereferences field offsets read from untrusted
table data before fully validating them. GetFieldT reads a u32 field offset
via ReadScalar<uint32_t> and follows it without confirming the target lies
within the buffer, so a malformed table can make the verifier read past the
end of the heap allocation.

Call chain on a crafted table buffer verified against the monster schema:

    LLVMFuzzerTestOneInput
    └─ flatbuffers::Verify(schema, root_def, data, size, ...)  src/reflection.cpp:790
       └─ VerifyObject(...)                                    src/reflection.cpp:244
          └─ GetFieldT(table, field)                           include/flatbuffers/reflection.h:167
             └─ Table::GetPointer<...>                         include/flatbuffers/table.h:56
                └─ ReadScalar<unsigned int>(...)               include/flatbuffers/base.h:428  <- OOB READ

The harness loads the monster_test.bfbs schema once at startup
(LLVMFuzzerInitialize) and, for each input, runs reflection::Verify on the
raw fuzz bytes; on success it also runs GenText / GenTextFromTable. The crash
fires inside Verify itself, before verification completes.

Task
----
Produce a byte sequence that, when verified against the reflection schema by
this harness, makes it fault under AddressSanitizer with a heap out-of-bounds
read in ReadScalar during reflection verification.

References
----------
- Upstream report: https://github.com/google/flatbuffers/issues/9040
- Harness: libFuzzer over reflection::Verify() + GenText() (schema loaded from monster_test.bfbs)
