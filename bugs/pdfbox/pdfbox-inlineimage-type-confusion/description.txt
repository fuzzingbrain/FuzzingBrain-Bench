Type Confusion DoS in PDInlineImage.getDecode
=============================================

Project:     pdfbox (pdmodel inline image)
Bug class:   type-confusion (ClassCastException)
Sanitizer:   JVM (uncaught exception)

PDInlineImage.getDecode() reads the inline image's `/D` (Decode) entry
with `(COSArray) parameters.getDictionaryObject(COSName.D, COSName.DECODE)`
— an unchecked cast to COSArray with no COS type check. When the `/D`
entry is a different COS object (e.g. a COSInteger rather than a COSArray)
the cast throws ClassCastException, an unhandled type confusion. The
harness ignores its byte input and unconditionally constructs a
PDInlineImage whose `/D` entry is set to an integer, then calls
getDecode() to drive the faulty cast.
