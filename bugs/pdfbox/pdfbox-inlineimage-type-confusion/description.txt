Type Confusion DoS in PDInlineImage.getDecode
=============================================

Project:     pdfbox (pdmodel inline image)
Bug class:   type-confusion (ClassCastException)
Sanitizer:   JVM (uncaught exception)

The harness builds a PDF inline image from the input and calls into
PDInlineImage. getDecode() reads the inline image's `/D` (Decode) entry
and casts it to a COSArray without checking its actual COS type. An
inline image whose `/D` entry is a different COS object (e.g. a name or
number rather than an array) triggers a ClassCastException, an unhandled
type confusion.
