Memory Leak in ElfLinker::addLoader() when packing ELF executables
==================================================================

Project:     upx (src/linker.cpp)
Bug class:   memory-leak
Sanitizer:   LeakSanitizer

The harness writes the input to a temp file and runs upx's pack path on
it as an ELF executable. ElfLinker::addLoader() duplicates the
section-name token list with `strdup` into a local `begin`
(src/linker.cpp:369), to be freed unconditionally at the end of the
function (linker.cpp:421). For a malformed input whose stub references an
unknown section, `findSection(sect)` (linker.cpp:394, called with the
default `fatal=true`) throws InternalError. That exception unwinds the
stack past the `::free(begin)` at the end, so the `strdup` buffer is
never freed and leaks. The harness's `catch(...)` lets the process exit
cleanly, and LeakSanitizer reports the leak at exit.
