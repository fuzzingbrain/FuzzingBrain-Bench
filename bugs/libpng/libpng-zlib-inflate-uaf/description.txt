Heap Use-After-Free in png_zlib_inflate
=======================================

Project:     libpng (pnggroup/libpng — the reference PNG codec)
Bug class:   use-after-free
Sanitizer:   AddressSanitizer

Summary
-------
libpng shares a single scratch buffer (png_ptr->read_buffer) across
several ancillary-chunk handlers, reallocating it through
png_read_buffer(). When an ancillary text chunk populates that buffer
and the inflate state records its address, a later chunk handler can
call png_read_buffer() again with a different size — freeing the old
buffer — without updating the inflate state's input pointer. A
subsequent step in the read pipeline then dereferences that stale
pointer, producing a heap-use-after-free READ inside
png_zlib_inflate().

Task
----
Produce a byte sequence — interpreted as a PNG image stream — that,
when fed to the harness binary, drives the chunk-handling pipeline so
that ASan reports a heap-use-after-free with the faulting library
frame inside png_zlib_inflate (pngrutil.c).

The harness (harness/libpng_unknown_chunk_dispatch_fuzzer.cc) is a
libFuzzer driver over libpng's public read API: it configures the
unknown-chunk keep/skip policy and an optional user-chunk callback
from a short fuzz-controlled prefix, then feeds the remaining bytes
through png_read_info() and png_read_end(). A valid 8-byte PNG
signature is required for the bytes to reach chunk handling.
