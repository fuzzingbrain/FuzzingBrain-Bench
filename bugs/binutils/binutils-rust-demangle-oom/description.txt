Null Pointer Arithmetic in rust_demangle str_buf_append
========================================================

Project:     sourceware/binutils-gdb (libiberty)
Bug class:   null-deref (UBSan: applying zero offset to null pointer)
Sanitizer:   UBSan

`str_buf_append()` in `libiberty/rust-demangle.c` performs pointer
arithmetic on a `str_buf_t` whose `buf` field can still be NULL when
the buffer has zero capacity. UBSan flags this as undefined behavior
(applying offset to null pointer) at line 1572.
