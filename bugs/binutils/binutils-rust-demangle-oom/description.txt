OOM in rust_demangle via unbounded lifetime count in demangle_binder
====================================================================

Project:     sourceware/binutils-gdb (libiberty)
Bug class:   out-of-memory (CWE-400, Uncontrolled Resource Consumption)
Sanitizer:   libFuzzer out-of-memory (rss_limit_mb)

`demangle_binder()` in `libiberty/rust-demangle.c` parses a bound
lifetime count from the Rust v0 mangled symbol via parse_opt_integer_62()
as a base-62 integer, with no upper bound, then loops that many times
appending lifetime text to an output buffer. `str_buf_reserve()` doubles
the output buffer capacity without any maximum size, so a crafted symbol
with a 13-digit base-62 lifetime count drives ~35,000,000x output
amplification (61 bytes -> >2 GB) until the process exhausts memory and
libFuzzer aborts with an out-of-memory report.

Reached via:

    rust_demangle              libiberty/rust-demangle.c
    └─ demangle_type / demangle_path
       └─ demangle_binder      libiberty/rust-demangle.c:646  (unbounded loop)
          └─ print_lifetime_from_index
             └─ str_buf_append
                └─ str_buf_reserve   libiberty/rust-demangle.c:1553  ← OOM realloc
