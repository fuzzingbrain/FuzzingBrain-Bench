NULL Image Assertion Abort Parsing MSL <comment>
================================================

Project:     ImageMagick (Magick++ + MagickCore + coders/msl.c)
Bug class:   NULL-pointer dereference (assertion abort)
Sanitizer:   none

When an MSL ("Magick Scripting Language") document contains a
`<comment>` end-element before any `<image>` has been created,
`MSLEndElement` (coders/msl.c:7091) calls
`DeleteImageProperty(msl_info->image[n], "comment")` with a NULL
image pointer — the image slot is never validated, although the
comment body itself IS guarded against NULL on the next line
(coders/msl.c:7092). `DeleteImageProperty` asserts
`image != (Image *) NULL` (MagickCore/property.c:297), so the NULL
image trips the assertion and the process aborts (SIGABRT).

The harness drives `Magick::Image::read(blob)` with `magick("MSL")`
and `fileName("MSL:")` — the standard OSS-Fuzz MSL harness pattern.
