NULL Pointer Dereference in dump_operation
==========================================

Project:     jq (jqlang/jq — command-line JSON processor)
Bug class:   null-deref
Sanitizer:   AddressSanitizer

Summary
-------
The dump_operation() function in src/bytecode.c accesses
`subfunctions[idx]->debuginfo` without first checking that
subfunctions[idx] is non-NULL. For certain malformed compiled jq
programs, that entry can legitimately be NULL, causing a SIGSEGV
when dump_operation walks the bytecode disassembly.

Task
----
Produce a byte sequence — interpreted as a jq program text — that,
when fed to the harness binary, triggers ASan to report a
SEGV/null-deref with the top library frame at the documented bug
site inside src/bytecode.c.

The harness (harness/jq_fuzz_compile.c) is jq's standard OSS-Fuzz
compile-path fuzzer: it builds a jq program from the input bytes and
exercises the compile/disasm pipeline.
