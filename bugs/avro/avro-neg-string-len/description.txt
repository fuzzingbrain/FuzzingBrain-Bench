NULL Pointer Member Access in Avro C Schema Union Parser
==========================================================

Project:     apache/avro (lang/c)
Bug class:   null-deref (UBSan: member access within null pointer)
Sanitizer:   UBSan

The Avro C schema parser's union-handling code dereferences an
internal `struct avro_union_schema_t*` without checking that it was
fully constructed. A union schema with certain malformed branch
references causes `avro_schema_union_append()` to UBSan-fail at
schema.c:392.

The harness exposes this indirectly: it picks a built-in schema using
the first input byte, then feeds the rest as a binary value to decode.
When the selected schema is a union, the parser path itself hits the
null deref before any value-reading work begins.

Call chain on the 2-byte PoC (selects union schema, then garbage):

    avro_schema_from_json_length     lang/c/src/schema.c:1285
    └─ avro_schema_from_json_root    lang/c/src/schema.c:1240
       └─ avro_schema_from_json_t    lang/c/src/schema.c:1157
          └─ avro_schema_union_append lang/c/src/schema.c:392  ← UBSan fires
