OOB Read in libsharpyuv Gamma-to-Linear Conversion
==================================================

Project:     libwebp / libsharpyuv (SharpYUV encode-side color conversion)
Bug class:   out-of-bounds read
Sanitizer:   AddressSanitizer

Summary
-------
libsharpyuv's gamma-to-linear conversion trusts every pixel value to fit
within the caller-declared rgb_bit_depth. On a high-bit-depth conversion
(rgb_bit_depth >= 10), a raw 16-bit sample whose value exceeds
(1 << rgb_bit_depth) - 1 is never clamped, and the unclamped value flows
into a fixed-point gamma-table interpolation whose computed table index
walks past the end of the static gamma-to-linear lookup table — an
out-of-bounds read.

Task
----
Produce a byte sequence that, when fed to the harness binary, drives
SharpYuvConvert with a high-bit-depth configuration and out-of-range
pixel samples so that the sanitizer reports an out-of-bounds read whose
faulting library frame is in libsharpyuv's gamma code
(sharpyuv/sharpyuv_gamma.c).

The harness (harness/sharpyuv_convert_fuzzer.cc) calls the public
SharpYuvConvert() API. Input layout: bytes 0-1 width seed, 2-3 height
seed, byte 4 selects rgb_bit_depth from {8,10,12,16}, byte 5 selects
yuv_bit_depth, bytes 8.. are the raw RGB pixel plane.

References
----------
- Source: sharpyuv/sharpyuv_gamma.c (FixedPointInterpolation / ToLinearSrgb)
