NULL Pointer Dereference in WebPMuxAssemble
===========================================

Project:     libwebp (webmproject/libwebp — WebP codec, mux API)
Bug class:   null-deref
Sanitizer:   AddressSanitizer

Summary
-------
WebPMuxCreate() accepts a malformed animated WebP whose frame is missing
its ANMF header chunk, leaving the internal frame's header pointer
(wpi->header) NULL. When the muxer later assembles the container,
GetFrameInfo() reads frame_chunk->data->size (src/mux/muxedit.c:407)
through that NULL header chunk pointer without checking it, causing a
NULL pointer dereference (SIGSEGV) in the muxer's frame-info path.

Task
----
Produce a byte sequence — interpreted as a WebP file — that, when fed
to the harness binary, drives WebPMuxAssemble() to dereference the NULL
frame header chunk pointer, making the sanitizer report a null-deref /
SEGV whose faulting library frame is in libwebp's muxer
(src/mux/muxedit.c).

The harness (harness/webp_mux_fuzzer.c) uses only the public webp/mux.h
API: it builds a WebPMux from the input bytes (WebPMuxCreate), queries
frames/chunks, sets a chunk, and finally calls WebPMuxAssemble().
